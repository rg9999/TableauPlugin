Key Design Elements

  • Left-edge color stripes: Blue = Navigation, Orange = Flight Control, Green = System Log. Instant visual scanning.
  • Sparse cells: Blank cells are truly empty (no "N/A" clutter). Only the message type's own fields have values.
  • Timestamp pinned left with subtle blue tint — always the anchor column.
  • Green-tinted rows at bottom = newly arrived data from live refresh.
  • Selected row (blue highlight) shows cmd_position at 1.0000 in red — data-level emphasis for the saturated actuator.
  • Status bar: Compact, always visible. Row count, message type count, new data count, last refresh time.

Empty State Design

  • Minimal, not empty: A simple directional hint — no tutorial, no wizard, no multi-step onboarding.
  • Tree is pre-populated: All ~200 message types are loaded from the Tableau data source. The analyst sees what's available immediately.
  • Status bar confirms readiness: "No fields selected" / "Ready" — the tool is waiting for the analyst, not loading.
  • First-time success target: Analyst should drag a field and see data within 30 seconds.

Row Detail Panel Design

  • Bottom panel (Wireshark-style): Opens below the grid, preserving grid context. Analyst can see both the grid row and all message fields.
  • Full message tree: Shows ALL fields (not just selected columns), with expand/collapse per nested group.
  • Name-value layout: Two-column layout (field name left, value right) for easy scanning.
  • Close button: Click ✕ to dismiss and reclaim grid space.
  • Resizable: Drag handle on top edge to resize (not shown in static mockup).

Collapsed Tree — Maximum Data Density

  • 32px icon strip: "▶ Fields" label in vertical text. One click to re-expand.
  • Full-width grid: All horizontal space dedicated to data columns.
  • More columns visible: With tree collapsed, additional fields (altitude_msl, heading) are visible without horizontal scrolling.
  • Analyst workflow: Set up fields with tree open → collapse tree → focus on data analysis.
